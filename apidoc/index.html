PrysmisAI API

A single OpenAI-compatible endpoint powered by you.com that routes to multiple top AI models. No API keys required. No sign-up. Just send requests and get responses.

Fully OpenAI Chat Completions compatible. Change the base URL in any existing OpenAI client and it works immediately.

Endpoint

One endpoint handles all chat completions:

POST https://api.prysmisai.wtf/v1/chat/completions

Model can also be passed as a query parameter:

POST https://api.prysmisai.wtf/v1/chat/completions?model=psm-v1.0

Request Format

Send a JSON body. All fields:

FieldTypeRequiredDefaultDescription
modelstringoptionalpsm-v1.0Model ID to use
messagesarrayrequired-Conversation history
temperaturenumberoptional0.7Randomness 0-2
max_tokensnumberoptional4096Max output tokens

Message object shape:

FieldValuesDescription
rolesystem user assistantWho authored the message
contentstring or arrayText content. Use an array for vision (image + text)

Available Models

Pass model ID in the model field. All models are hosted locally via Ollama.

Available Models
psm-v1.0 PSM-v1.0(PrysmisAI) PrysmisAI model hosted locally via Ollama

PSM-v1.0(PrysmisAI) is hosted locally via Ollama. Make sure Ollama is running with llama3.2-vision:latest installed.

Examples

JavaScript

JS 
const response = await fetch('https://api.prysmisai.wtf/v1/chat/completions', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    model: 'psm-v1.0',
    messages: [
      { role: 'system', content: 'You are a helpful assistant.' },
      { role: 'user', content: 'Write me a Roblox health bar script.' }
    ],
    max_tokens: 4096
  })
});

const data = await response.json();
console.log(data.choices[0].message.content);

Python

PY 
import requests

res = requests.post('https://api.prysmisai.wtf/v1/chat/completions', json={
    'model': 'psm-v1.0',
    'messages': [
        {'role': 'user', 'content': 'Create a Roblox shop system.'}
    ],
    'max_tokens': 4096
})

print(res.json()['choices'][0]['message']['content'])

cURL

SH 
curl -X POST https://api.prysmisai.wtf/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "psm-v1.0",
    "messages": [{"role": "user", "content": "Hello!"}],
    "max_tokens": 2048
  }'

Response Format

Standard OpenAI Chat Completions response:

JSON 
{
  "id": "chatcmpl-...",
  "object": "chat.completion",
  "model": "gpt-5.2",
  "choices": [
    {
      "index": 0,
      "message": {
        "role": "assistant",
        "content": "Response text..."
      },
      "finish_reason": "stop"
    }
  ],
  "usage": {
    "prompt_tokens": 24,
    "completion_tokens": 180,
    "total_tokens": 204
  }
}

Get the text: data.choices[0].message.content

Errors

StatusWhen
400Missing or invalid messages array
500All model fallbacks exhausted

Error body:

{ "error": "No valid messages provided" }

Plugin API (Studio Integration)

These endpoints connect your Roblox Studio plugin to the website. The plugin polls for changes and sends them into your game live.

POST /plugin/ping

Verifies plugin token and marks plugin as connected.

FieldTypeDescription
pluginTokenstringPlugin token from Studio widget
{ "ok": true, "model": "psm-v1.0", "user": "username" }

GET /plugin/poll?pluginToken=

Called every 0.6s by the plugin to receive pending changes and commands from the website.

{ "changes": [{ "id": "abc123", "code": "..." }], "command": null, "model": "psm-v1.0" }

POST /plugin/execute

Queues a code change or JSON command to be applied in Studio. Called when user clicks "Apply Change" on the website.

FieldTypeDescription
codestringLua code to apply (use this OR command)
commandobjectJSON command object (use this OR code)
descriptionstringOptional label for the change

JSON command types: create_script, create_part, create_model, create_folder, create_remote, set_property, delete, rename, move, clone, batch

JSON
{
  "command": {
    "type": "create_script",
    "path": "ServerScriptService > MyScript",
    "code": "print('Hello from PrysmisAI')",
    "scriptType": "Script"
  }
}

POST /plugin/ack

Sent by plugin after applying a change, reporting success/failure back to the website.

FieldDescription
pluginTokenPlugin auth token
changeIdID of the change that was applied
oktrue if applied successfully
resultsArray of per-script result objects

POST /plugin/files

Plugin sends current game file tree to website so the AI has context about your game structure.

GET /plugin/files

Website retrieves the latest game file tree sent by the plugin. Requires user auth token.

Getting Started

Setup Tutorial

Follow these steps to start using the PrysmisAI API in your own project.

1
Create your account

Go to prysmisai.wtf and sign up for a free account. No credit card needed.

2
Make your first request

Copy this into your project. Replace the message content with whatever you want to ask.

JS
const res = await fetch('https://api.prysmisai.wtf/v1/chat/completions', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    model: 'psm-v1.0',
    messages: [{ role: 'user', content: 'Hello!' }]
  })
});
const data = await res.json();
console.log(data.choices[0].message.content);
3
Choose your model

Use psm-v1.0 as the model. The PSM-v1.0(PrysmisAI) model is hosted locally via Ollama.

4
Add a system prompt

Guide the AI's behavior by adding a system message as the first item in your messages array.

JS
messages: [
  { role: 'system', content: 'You are a Roblox game development expert.' },
  { role: 'user', content: 'Write me a health bar script.' }
]
5
Handle the response

The AI text is always at data.choices[0].message.content. Check data.choices[0].finish_reason - if it is "length" the response was cut off and you can request a continuation.

6
Connect the Roblox Studio plugin

Install the official plugin from the Roblox Creator Store. Open it in Studio, paste your plugin token from your dashboard Settings, and click Connect. The website and Studio are now linked - accepted AI changes run in your game instantly.
